rawgithub.com

The Problem

You've got a raw.github.com URL that points to some work-in-progress HTML, JavaScript, or CSS. You'd like to share it with someone for a quick demo, or maybe you'd like to use it in a jsPerf test.

Too bad! Most things served from raw.github.com have a text/plain content type, which means HTML won't render, and some browsers won't execute JavaScript or apply CSS.

The Solution

Delete the dot! Just replace raw.github.com with rawgithub.com and skadoosh, you're loading raw files from your GitHub repo and they're served with appropriate content types. Render HTML, apply CSS, execute JavaScript.

Before

https://raw.github.com/user/repo/master/filename.js

After

https://rawgithub.com/user/repo/master/filename.js

It works with gists too! Just replace gist.github.com with rawgithub.com in the raw gist URL.

The Caveat

Use this for testing and for sharing demo code with friends, but don't use it for anything production-related. It's slow, it might break from time to time, and it's intended as a convenience for development, not free static hosting.

If you send production traffic to rawgithub.com, evil things will happen to your site. You've been warned!

The FAQ

Can I use rawgithub.com on a production website?

No. Don't do this. Not even a little bit. It will make me very sad.

rawgithub.com is a free service to make developers' lives easier when developing and testing and debugging things. It's not free hosting. Use GitHub Pages for that.

What will happen if I hotlink to a rawgithub.com URL on a production website?

If you hotlink to a rawgithub.com URL on a site that gets lots of traffic, requests will be throttled. If this heavy traffic continues, requests originating from your site will eventually be dropped.

If the traffic continues even after automatic throttling is triggered, rawgithub.com will start serving evil.js and evil.css instead of requested JS and CSS files. Visitors to your site may see an alert warning against using rawgithub.com in production, and your JS may start doing very strange things.

I tried to claim a rawgithub.com URL on Google Webmaster Central and rawgithub.com returns a 403 error instead of my verification HTML. What gives?

You can't claim rawgithub.com URLs on Google Webmaster Central, because rawgithub.com isn't a hosting service. It would be pointless anyway, since all rawgithub.com responses include an X-Robots-Tag: none header, which tells search crawlers not to index anything.
